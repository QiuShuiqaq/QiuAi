# QiuAi 响应式桌面布局调整设计

## 背景

当前 `QiuAi` 桌面前端在窗口缩放时存在以下问题：

- `单图设计` 的 `效果展示` 在窗口缩小时会过早塌缩为单列，视觉上像“只显示一张图”。
- 参数设置区、效果展示区、结果导出区、任务侧栏主要依赖固定列数和少量断点切换，缩放过程不连续。
- 多个卡片和图片区使用固定 `grid-template-columns`，导致窗口变化时组件不能平滑跟随尺寸变化。

本次目标是在不改变现有主业务结构的前提下，让主界面在桌面窗口缩放时保持连续、稳定、清晰的响应式表现。

## 目标

1. 左侧菜单中 `单图设计` 改名为 `单图测试`。
2. `效果展示` 图片结果区域改为流式自适应布局，窗口缩放时平滑变化。
3. 主工作区的参数区、展示区、右侧栏、任务卡片、导出卡片同步支持柔性缩放。
4. 保持桌面应用固定主结构，不改成移动端式页面，不引入新的业务模块。
5. 不修改后端任务逻辑，仅调整前端布局、文案标签和相关前端测试。

## 设计方案

### 1. 菜单和模块命名

- 左侧菜单 `单图设计` 全量改名为 `单图测试`。
- 与该菜单对应的展示标题、空状态文案、测试断言同步更新。
- 内部业务 key 继续保留 `single-image`，避免影响后端任务链与本地数据结构。

### 2. 主工作区布局策略

保留现有桌面结构：

- 左侧导航栏
- 中间主工作区
- 右侧侧边栏

主工作区在非 `工作台 / 模型价格` 页面继续保持“参数设置 + 效果展示”双区结构，但改为柔性比例布局：

- 默认维持左窄右宽的视觉关系
- 不再依赖过强的固定列宽感
- 在中等宽度下优先压缩间距、内边距、卡片高度
- 仅在更窄窗口时才切换成单列堆叠

### 3. 效果展示区策略

`效果展示` 区是本次核心。

#### 单图测试

- 现有四模型结果卡片从固定三列/两列/一列，改为 `auto-fit + minmax(...)` 的流式网格。
- 每张图卡片：
  - 宽度跟随容器变化
  - 图片维持稳定比例
  - 模型名允许换行或截断保护
- 在宽窗口下尽量展示更多列，在中小窗口下平滑减列，而不是突然塌到 1 列。

#### 套图设计 / 套图生成

- 每组结果里的图片网格也改为同样的流式自适应策略。
- 组卡片本身缩小内边距和间距，避免窗口缩小时卡片内容过早拥挤。
- 总结信息区 `summary-metrics` 也同步使用可收缩网格，而不是固定块宽。

### 4. 侧边栏与右侧栏策略

#### 左侧菜单

- 按钮高度、文字大小、内边距改为随断点缩放。
- 文本保持居中，避免小窗口时视觉失衡。

#### 右侧栏

- `任务队列` 卡片尺寸继续紧凑化，但不允许文本、进度条、翻页控件互相挤压。
- `结果导出` 中的导出项、图标按钮、信息行也改为可收缩布局。

### 5. CSS 实现原则

本次前端布局调整遵循以下约束：

- 优先使用 `minmax(0, 1fr)`、`auto-fit`、`clamp(...)`、弹性 `gap/padding` 实现连续缩放。
- 避免只依赖单一固定断点硬切换。
- 所有主容器保持 `min-width: 0`、`min-height: 0`，防止内容把父层撑爆。
- 图片展示卡片不写死宽度，交给网格自动分配。
- 仅在必要的窄窗口断点下改为单列布局。

## 影响范围

主要涉及：

- `renderer/src/App.vue`
- `renderer/src/components/WorkspaceSidebar.vue`
- `renderer/src/components/DesignWorkspace.vue`
- `renderer/src/components/ResultDisplayPanel.vue`
- `renderer/src/components/TaskManagerSidebar.vue`
- `renderer/src/components/ResultExportPanel.vue`
- `renderer/src/assets/styles.css`
- 对应 renderer source tests

不涉及：

- Electron 主进程逻辑
- API 请求链路
- 本地数据存储结构
- 任务生成后端逻辑

## 验收标准

1. 左侧菜单显示 `单图测试`。
2. 缩放窗口时，单图测试的 4 张结果图不会异常提前塌成只有 1 张可视。
3. 套图设计、套图生成、结果导出、任务队列在缩放时无明显重叠、溢出、错位。
4. 页面布局随窗口宽度连续调整，观感自然。
5. `npm test`、`npm run lint`、`npm run build:renderer` 通过。

## 风险与处理

- 风险：大范围改样式可能影响工作台和模型价格页的既有视觉平衡。
- 处理：优先收敛在展示区、双栏区、右侧栏和断点样式，工作台结构不重做，只做必要联动。
